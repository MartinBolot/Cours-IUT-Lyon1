2) Qu’est ce qu’un gestionnaire de dépendance ? Maven est-il le seul ? Quel est l’avantage d’utiliser un
gestionnaire de dépendances ? Quelles sont les grandes étapes de fonctionnement de Maven ?
Un gestionnaire de dépendance permet de déclarer, gérer et partager les dépendances logicielles d'un projet. Maven n'est pas le seul outil à le faire.
L'avantage est de pouvoir découpler le code source de l'application du code tiers.
Cela facilite le processus de mise à jour dans le cadre de l'utilisation d'une bibliothèque de version par exemple.
Cela amélore la capacité du code à être sous gestionnaire de version.
Maven : Déclaration de la dépendance -> installation du projet
